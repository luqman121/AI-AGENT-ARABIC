# M2 Layer C — Sandbox and Static Website Artifacts

**Status:** Approved for implementation on 2026-07-18.

## 1. Outcome and vertical slice

Layer C adds the smallest complete artifact flow after Layer B: a user reviews the saved Arabic
plan, explicitly starts a separate execution run, and receives a validated static website preview
plus a downloadable ZIP. The generated website is a single self-contained `index.html` so private
preview access remains reliable without making related assets public.

This slice supports static websites only. PDF, spreadsheet, presentation, image, game templates,
publishing, custom domains, and external messaging remain later work.

## 2. Product flow

1. A `planning` run continues to produce the reviewed Arabic plan from Layer B.
2. After a successful plan, the run panel offers an explicit Arabic “start implementation” action.
3. The web app creates an idempotent `execution` run linked to the latest successful planning run.
4. The worker generates a bounded, schema-validated static-site manifest from the user request and
   reviewed plan. Raw generated JSON is not presented as progress.
5. The worker creates a private, ephemeral Daytona sandbox with outbound networking blocked, writes
   the generated HTML and a trusted validator there, and executes only fixed validation commands.
6. After validation, the worker uploads the HTML preview and ZIP to private Cloudflare R2 through
   its S3-compatible API, persists artifact metadata, and finishes the run transactionally.
7. The tenant-authorized preview page issues short-lived signed URLs. Preview HTML runs on the R2 S3
   API origin in a sandboxed iframe; the control-plane origin never executes generated code.

## 3. Run and persistence model

- Add `runs.kind` with `planning | execution` and `runs.parent_run_id` for the reviewed plan link.
- Add `sandbox_provider`, `sandbox_id`, and `sandbox_duration_ms` accounting fields.
- Add an `artifacts` table with tenant-preserving foreign keys to workspace, project, and run;
  immutable private object keys; media type, byte size, checksum, kind, and creation time.
- Add execution event types for generation, sandbox creation/validation, upload, and artifact-ready.
- Continue enforcing one active run per project across both run kinds.
- An execution run may start only when the same workspace/project has a successful planning run with
  a saved assistant plan and no newer user message after that plan.

## 4. Generated-content contract

The versioned `static-site.ar.v1` prompt keeps system, developer, and user/reviewed-plan content
separate. Its JSON response contains:

- `summary`: short Arabic completion text;
- `html`: one complete UTF-8 HTML document.

Validation rejects responses that are not valid JSON, exceed configured byte limits, omit Arabic
language/RTL metadata, contain remote URLs, forms, navigation away from the artifact, or unsupported
schemes. A strict CSP meta tag is required or injected before sandbox validation. Prompt-injection,
external-network, oversized-output, malformed-document, and unsafe-side-effect eval cases are
included.

## 5. Sandbox boundary

`packages/sandbox` owns the Daytona SDK and exposes only provider-neutral validated execution
contracts. The worker cannot import the Daytona SDK directly.

Every sandbox is:

- private and ephemeral;
- created with `networkBlockAll: true`;
- assigned a hard TTL and per-command timeout;
- given no model, database, Redis, S3, auth, or application credentials;
- labelled only with non-secret run metadata;
- deleted in `finally`, with TTL as the recovery backstop.

The adapter returns bounded validation metadata and generated bytes, never arbitrary logs. Sandbox
failures map to stable redacted error codes.

## 6. Artifact storage and access

`packages/artifacts` owns Cloudflare R2 upload and signing behavior through AWS SDK v3 because both
worker and web need the same private S3-compatible object boundary. Objects use unpredictable
tenant/run/artifact UUID keys and never become public. Uploads set exact content type, content
disposition, checksum metadata, and bounded content length.

The preview page first verifies workspace membership and project/artifact ownership, then signs the
preview and download objects for at most five minutes. Signed URLs are never stored in PostgreSQL,
logs, events, or client persistence. The iframe uses a separate R2 S3 API origin and
`sandbox="allow-scripts"` without `allow-same-origin`.

## 7. Limits and failures

Layer C adds configured maximums for generated HTML bytes, ZIP bytes, sandbox wall time, command
time, and artifact count. Existing model time/token/attempt/spend limits still apply to execution.
Cancellation is checked before model generation, before sandbox creation, after validation, and
before upload. No retry occurs after an artifact upload without an idempotent object key.

Stable failures distinguish missing execution prerequisites, unsupported artifact, invalid generated
content, sandbox unavailable/timeout/validation failure, artifact too large, storage unavailable,
and cancellation. Arabic UI copy remains provider-neutral.

## 8. Verification

- Unit: static-site schema/security checks, ZIP/checksum creation, sandbox error mapping, and env
  validation without values.
- Adapter: Daytona calls against an injected fake client proving private/ephemeral/network-blocked
  creation, fixed commands, bounded downloads, and cleanup.
- Integration: migration constraints; tenant-safe execution start; one active run; reviewed-plan
  prerequisite; one artifact/final message under retries; private storage upload/signing.
- Browser: planning remains review-first; execution events are persisted; completed preview and ZIP
  actions appear; cross-tenant artifact IDs stay hidden; cancellation/failure/empty states work at
  `390x844` and `430x932`.
- Required repository gates: format, lint, strict typecheck, unit, migration, integration, build,
  functional Playwright, and visual Playwright.

No production sandbox or storage request is required for deterministic CI. A separately authorized
live smoke test may use the configured private credentials after all local gates pass.
